# Phase 6 Extension Coverage

Reference target: PHP 8.5.7 (`php-8.5.7`).

| Extension or area | Phase 6 status | Required gate |
| --- | --- | --- |
| Core constants and functions | Introspection scaffold plus generated function/class/constant coverage report against `REFERENCE_PHP` when configured | `test-phase6`, `diff-stdlib`, `coverage-phase6` |
| Standard variable/type and debug functions | Implemented for scalar, array, object, resource, reference, conversion, and recursion-safe debug-output cases; byte-perfect zval dumps tracked as `PHASE6-GAP-DEBUG-BYTE-PERFECT-ZVALS` | `diff-stdlib` |
| Serialization | MVP implemented for scalars, arrays, simple objects, malformed-input safety with PHP-style warnings, and bounded parsing; reference records, `allowed_classes`, magic hooks, and resources are known gaps | `diff-stdlib` |
| String length/search/compare | Binary-safe MVP implemented for `strlen`, `substr`, `strpos`, `stripos`, `strrpos`, `str_contains`, `str_starts_with`, `str_ends_with`, `strcmp`, `strncmp`, `strcasecmp`, and `strncasecmp` | `diff-stdlib` |
| String split/join/replace/trim/case/pad | Byte-oriented MVP implemented for `explode`, `implode`, `str_replace`, `strtr`, trim variants, ASCII case helpers, repeat, pad, and reverse; locale-specific behavior is stabilized to ASCII/C-locale semantics | `diff-stdlib` |
| String encoding/hash/HTML/URL helpers | MVP implemented for hex/base64/hash/CRC, `hash`, `hash_hmac`, `ord`/`chr`, common HTML escaping, URL encode/decode, and `http_build_query`; malformed `hex2bin` returns `false` with PHP-style warnings; full entity flags, the full PHP hash algorithm registry, and query options are known gaps | `diff-stdlib` |
| Hash and random extensions | Optional Phase 6 hash/random MVP implemented for `hash`, `hash_hmac`, `random_bytes`, and `random_int`; hash algorithms are limited to documented MD5/SHA1/CRC32 MVP coverage and random tests assert shape/range rather than fixed bytes | `diff-stdlib` |
| Formatted output | MVP implemented for `printf`, `sprintf`, `vprintf`, and `vsprintf`; stream-backed `fprintf` and advanced locale/positional formats are known gaps | `diff-stdlib` |
| Array basics and access | MVP implemented for `count`, `sizeof`, key/value/list helpers, strict/loose search, and `array_column` over array rows | `diff-stdlib` |
| Array stack/slice/merge | MVP implemented for `array_push`, `array_pop`, `array_shift`, `array_unshift`, `array_slice`, `array_splice`, merge/replace/reverse/pad/chunk/flip helpers; `array_flip` emits PHP-style skipped-entry warnings; recursive-reference edge cases are known gaps | `diff-stdlib` |
| Array callbacks | MVP implemented for `array_map`, `array_filter`, `array_reduce`, `array_walk`, `array_any`, `array_all`, `array_find`, and `array_find_key`; `array_walk` callback-side by-reference mutation is a known gap | `diff-stdlib` |
| Array sorting | MVP implemented for `sort`, `rsort`, `asort`, `arsort`, `ksort`, `krsort`, `usort`, `uasort`, `uksort`, `natsort`, and `natcasesort`; natural-sort edge cases are a known gap | `diff-stdlib` |
| Math and numeric functions | MVP implemented for `abs`, `min`, `max`, `round`, `floor`, `ceil`, `sqrt`, `pow`, `intdiv`, `fmod`, `is_finite`, `is_infinite`, `is_nan`, and `number_format`; advanced rounding modes and float edge formatting are known gaps | `diff-stdlib` |
| Symbol introspection and function/object handling | Implemented for `defined`, `constant`, `function_exists`, `class_exists`, `interface_exists`, `enum_exists`, `method_exists`, `property_exists`, `is_subclass_of`, `get_class`, `get_parent_class`, `get_declared_classes`, `get_declared_interfaces`, `get_object_vars`, `get_mangled_object_vars`, `get_class_methods`, `get_class_vars`, `call_user_func`, `call_user_func_array`, `forward_static_call`, `func_get_args`, `func_num_args`, and `func_get_arg`; trait declaration introspection, full forward-static edge parity, and by-reference callable-array argument propagation are tracked known gaps | `diff-stdlib` |
| INI/config | MVP implemented for `ini_get`, `ini_set`, `ini_get_all`, and `get_cfg_var` over documented deterministic entries; `include_path` drives include resolution | `diff-stdlib` |
| Error handling | MVP implemented for `error_reporting`, `set_error_handler`, `restore_error_handler`, `trigger_error`, `user_error`, `set_exception_handler`, and `restore_exception_handler`; trigger/user errors use request-local handler stacks, `error_reporting`, `display_errors`, and runtime diagnostics | `diff-stdlib` |
| Output buffering | MVP implemented for `ob_start`, `ob_get_contents`, `ob_get_clean`, `ob_get_length`, `ob_get_level`, `ob_end_clean`, `ob_end_flush`, and `flush`; callbacks are tracked as `PHASE6-GAP-OUTPUT-BUFFER-CALLBACKS` | `diff-stdlib` |
| Environment and CLI superglobals | MVP implemented for controlled `getenv`, `putenv`, `$_ENV`, deterministic CLI `$_SERVER`, `php_sapi_name`, `php_uname`, and `get_current_user`; host environment and host identity are not imported implicitly | `diff-stdlib` |
| Process and OS capability surface | Prompt 06.48 registers `proc_open`, `proc_close`, `proc_get_status`, `popen`, `pclose`, `shell_exec`, `exec`, `passthru`, and `system`. The default runtime returns deterministic PHP-visible failures and process-capability warnings without launching a shell; isolated VM tests can mock output for shell-like APIs. Real process resources and host execution are tracked as `PHASE6-GAP-PROCESS-CAPABILITY`. | `process-capability-smoke` |
| Remaining arrays | Additional specialized helpers are outside prompts completed so far | `diff-stdlib` |
| Streams and filesystem | Resource handle model, request-local `ResourceTable`, metadata, close/finalization lifecycle, resource type helpers, root-constrained local file wrapper, `php://memory`/`php://temp` buffers, path helpers, stat MVP, PHP-visible local file read/write/handle functions, directory resources, `scandir`, local `glob`, `getcwd`, context-local `chdir`, stream metadata/content/copy helpers, stream contexts, and include-path resolution are implemented; network streams, advanced glob flags, VM-wide CWD persistence, byte-perfect stat arrays/metadata, warning text, `fseek` whence, host TTY probing, and `tmpfile` lifetime are tracked known gaps | `diff-streams` |
| JSON | MVP implemented for `json_encode`, `json_decode`, `json_validate`, last-error helpers, common constants, `JsonException`, and `JsonSerializable` symbols; advanced flags, bigint preservation, VM-persistent last-error state, and userland `JsonSerializable` dispatch are known gaps | `diff-json-pcre-date` |
| PCRE | MVP implemented through PCRE2 for `preg_match`, `preg_match_all`, `preg_replace`, `preg_replace_callback`, `preg_split`, `preg_grep`, `preg_quote`, and last-error helpers; common delimiters, modifiers, captures, offsets, replacements, split/grep flags, and constants are covered; advanced PCRE flags, byte-perfect warning text, full JIT/backtracking parity, and userland callback dispatch are known gaps | `diff-json-pcre-date` |
| Date/Time | MVP implemented for timezone defaults/listing, `date`, `time`, restricted `strtotime`, `DateTimeInterface`, `DateTime`, `DateTimeImmutable`, `DateInterval`, and `DateTimeZone` metadata plus internal DateTime object helpers for constructor-style creation, formatting, timestamp/timezone mutation, interval add/sub, modify, and diff; PHP VM method dispatch, full timelib parsing, timezone transitions, aliases, and warning/exception byte parity are known gaps | `diff-json-pcre-date` |
| SPL | Basis implemented for enabled extension metadata, SPL exception hierarchy catch/`instanceof`, `Traversable`/`Iterator`/`IteratorAggregate`/`ArrayAccess`/`Countable`/`Serializable` symbols, VM-owned autoload stack functions, stable `spl_object_id`/`spl_object_hash`, SPL iterator MVP objects for `ArrayIterator`, `RecursiveArrayIterator`, `IteratorIterator`, `LimitIterator`, `EmptyIterator`, and `AppendIterator`, SPL container MVP objects for `ArrayObject`, `SplFixedArray`, `SplObjectStorage`, `SplDoublyLinkedList`, `SplStack`, and `SplQueue`, and SPL file MVP objects for `SplFileInfo`, `SplFileObject`, and `SplTempFileObject`; full interface method enforcement, complete ArrayAccess/Serializable/file flag behavior, mutation/seek/recursive iterator edge cases, advanced autoload flags/path conventions, and Zend hash byte parity are known gaps | `diff-spl-reflection` |
| Reflection | MVP implemented for `ReflectionFunction`, `ReflectionMethod`, `ReflectionClass`, `ReflectionProperty`, `ReflectionClassConstant`, `ReflectionParameter`, `ReflectionNamedType`, `ReflectionEnum`, `ReflectionEnumUnitCase`, `ReflectionEnumBackedCase`, `ReflectionAttribute`, and `ReflectionExtension` over userland symbols plus Phase 6 internal registry symbols; internal functions/classes expose extension names, deterministic parameter/return metadata for smoke paths, enum backed cases and enum-case constants are queryable, member visibility/type/readonly/default metadata is visible, attributes expose arguments and repeat metadata, and internal file/line/doc values use PHP-like `false`. Full arginfo parity, complete internal method surfaces, extension versions/dependencies/INI entries, property-hook reflection object parity, and attribute instantiation parity are known gaps | `diff-spl-reflection` |
| Tokenizer | MVP implemented over the Phase 1 lexer for `token_get_all`, `token_name`, `PhpToken::tokenize`, `PhpToken::is`, `PhpToken::isIgnorable`, token constants, source text, line, and byte-position metadata; `TOKEN_PARSE` contextual behavior and Zend numeric token ID parity are documented gaps | `test-phase6` |
| Selected extension PHPT smoke | Prompt 06.51 adds `fixtures/phase6/phpt_extension_manifest.toml` and `phase6-phpt-smoke` for selected upstream PHPTs from `ext/standard`, `spl`, `json`, `pcre`, and `date`. The selector references php-src paths without vendoring PHPT contents, converts missing checkout/files into explicit skips, delegates runnable cases to the existing `run-phpt-smoke` harness, and writes a normalized report under `target/phase6/phpt-extension-smoke`. Broader upstream extension PHPT promotion is tracked as `PHASE6-GAP-EXTENSION-PHPT-PROMOTION`. | `phase6-phpt-smoke` |
| Regression corpus | Prompt 06.53 adds `tests/fixtures/phase6/corpus` and `phase6-corpus-smoke` for Composer/framework-adjacent snippets covering container autoload, env parsing, JSON config, regex routing, DateTime/version parsing, array manipulation, and reflection attributes without vendoring external frameworks. | `phase6-corpus-smoke` |
| Composer source-mode fixtures | Prompt 06.44 implements a local Composer-style PSR-4 autoload fixture where `require 'autoload.php'` resolves through `include_path`, the PHP autoload callback uses `stream_resolve_include_path`, `class_exists(..., true)` loads classes from `tests/fixtures/phase6/composer/project/src`, missing classes return safely, and autoloaded methods execute from their included unit. Prompt 06.45 adds deterministic `basic_project` generated-autoload fixtures for PSR-4, classmap, and files autoload metadata, regenerated by `composer-fixture-prepare` and covered by composer-area differential tests. Prompt 06.46 adds `composer-smoke-autoload` for files-before-classes order, PSR-4/classmap loading, repeated `include_once`, autoload stack count, and missing-class safety snapshots. Prompt 06.47 adds `composer-smoke-platform` for Composer-style `platform_check.php`, including `PHP_VERSION_ID`, `extension_loaded`, `get_loaded_extensions`, `ini_get`, `defined`, `constant`, `class_exists`, `function_exists`, `version_compare`, and absent-extension reporting. Prompt 06.48 adds a VM-only default-off process capability smoke so Composer-facing process probes fail safely without shell execution. Prompt 06.49 adds `composer-smoke-source`, an opt-in Composer source checkout runner that skips cleanly without `PHASE6_COMPOSER_SOURCE_DIR` and reports prioritized missing functions/classes on failure. | `composer-smoke`, `composer-smoke-autoload`, `composer-smoke-platform`, `process-capability-smoke`, `composer-smoke-source` |
| PHAR | ADR-0066 keeps PHAR out of required Phase 6 gates; Composer source mode is mandatory first, and any read-only PHAR MVP must be accepted as a later optional prompt before implementation | `coverage-phase6` |
| mbstring / intl / ICU | Out of required scope; optional stubs only | `coverage-phase6` |
