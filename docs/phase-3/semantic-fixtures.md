# Semantic Fixtures

Semantic fixtures are small PHP files that exercise one frontend rule or one
closely related group of rules. They are not corpus dumps, and each file should
prefer one focused acceptance or diagnostic expectation.

## Matrix

| Category | Purpose | Reference classification | Rust diagnostic IDs | Known gaps |
| --- | --- | --- | --- | --- |
| `valid` | accepted baseline files | accepted | none | none |
| `invalid` | parser or baseline rejects | rejected | parser diagnostics | none |
| `namespaces` | namespace forms and placement | accepted/rejected by placement | `E_PHP_MIXED_NAMESPACE_DECLARATIONS`, `E_PHP_NAMESPACE_MUST_BE_FIRST_STATEMENT` | none |
| `names` | imports and static name metadata | accepted except duplicate aliases | `E_PHP_DUPLICATE_USE_ALIAS` | runtime fallback lookup is metadata |
| `declarations` | declaration table | accepted plus semantic-only duplicate-class reject | `E_PHP_DUPLICATE_DECLARATION` | cross-file/include symbols deferred |
| `scopes` | lexical scope tree | accepted | none | runtime capture values deferred |
| `control_flow` | control context checks | accepted/rejected by context | `E_PHP_BREAK_NOT_IN_LOOP_OR_SWITCH`, `E_PHP_CONTINUE_NOT_IN_LOOP_OR_SWITCH`, `E_PHP_INVALID_BREAK_CONTINUE_LEVEL`, `E_PHP_RETURN_OUTSIDE_ALLOWED_CONTEXT`, `E_PHP_RETURN_VALUE_FROM_VOID_FUNCTION`, `E_PHP_RETURN_FROM_NEVER_FUNCTION`, `E_PHP_YIELD_OUTSIDE_FUNCTION`, `E_PHP_GOTO_LABEL_NOT_FOUND` | `goto-invalid-known-gap.php` |
| `types` | type lowering | accepted/rejected by type context | `E_PHP_INVALID_TYPE_VOID_CONTEXT`, `E_PHP_INVALID_TYPE_NEVER_CONTEXT`, `E_PHP_INVALID_TYPE_STATIC_CONTEXT`, `E_PHP_INVALID_TYPE_SELF_CONTEXT`, `E_PHP_INVALID_TYPE_PARENT_CONTEXT`, `E_PHP_INVALID_TYPE_CALLABLE_CONTEXT`, `E_PHP_DUPLICATE_TYPE_ALTERNATIVE` | runtime class existence deferred |
| `functions` | function-like signatures and flags | accepted/rejected by signature rule | `E_PHP_DUPLICATE_PARAMETER`, `E_PHP_VARIADIC_PARAMETER_NOT_LAST`, `E_PHP_INVALID_PARAMETER_DEFAULT`, `E_PHP_CLOSURE_USE_DUPLICATES_PARAMETER`, `E_PHP_DUPLICATE_CLOSURE_USE_VARIABLE`, `E_PHP_RETURN_VALUE_FROM_VOID_FUNCTION`, `E_PHP_RETURN_FROM_NEVER_FUNCTION` | generator runtime details deferred |
| `classes` | class-like HIR and checks | accepted/rejected by class rule | `E_PHP_DUPLICATE_CLASS_MEMBER`, `E_PHP_INVALID_PROPERTY_PROMOTION`, `E_PHP_INVALID_CLASS_CONTEXT_NAME`, `E_PHP_INVALID_MAGIC_METHOD_SIGNATURE` | autoload/runtime inheritance deferred |
| `interfaces` | interface declarations | accepted | none | inherited compatibility deferred |
| `traits` | trait use and adaptations | accepted/rejected by adaptation shape | `E_PHP_TRAIT_ADAPTATION_INVALID_SHAPE` | composition conflict resolution deferred |
| `enums` | enum cases and backing metadata | accepted/rejected by enum rule | `E_PHP_ENUM_CASE_VALUE_ON_UNIT_ENUM`, `E_PHP_ENUM_CASE_MISSING_VALUE_ON_BACKED_ENUM`, `E_PHP_DUPLICATE_CLASS_MEMBER` | backing value compatibility deferred |
| `attributes` | attribute metadata and args | accepted/rejected by const expr args | `E_PHP_ATTRIBUTE_ARGUMENT_NOT_CONST_EXPR` | attribute instantiation deferred |
| `const_expr` | constant-expression candidates | accepted/rejected by structural form | `E_PHP_INVALID_CONST_EXPR`, `E_PHP_ATTRIBUTE_ARGUMENT_NOT_CONST_EXPR` | value evaluation deferred |
| `declare` | file-level declare metadata | accepted/rejected for strict_types | `E_PHP_INVALID_STRICT_TYPES_DECLARE`, `E_PHP_STRICT_TYPES_DECLARE_NOT_FIRST` | runtime directive effects deferred |
| `deferred` | include/require/eval boundary | accepted by lint | `N_PHP_RUNTIME_CHECK_DEFERRED` | runtime effects intentionally deferred |
| `php85` | focused PHP 8.5 accepted forms | accepted by pinned PHP 8.5.7 | none | runtime/value semantics deferred |

Additional current categories `expressions`, `statements`, and `modifiers`
document structural HIR and modifier validation work that predates the final
matrix split.

Each category has a local `README.md` with purpose, example rules, expected
reference classification, expected Rust diagnostic IDs, and known gaps.

## Snapshot Convention

Deterministic snapshots live under `fixtures/semantic/snapshots/` and are
generated by:

```bash
nix develop -c scripts/run_semantic_fixtures.py --write-snapshots
```

The filename is the fixture path with `/` replaced by `__` and `.php` replaced
by `.snap`.

Snapshot format:

```text
source: path
parse_ok: bool
semantic_ok: bool
diagnostics:
  - id: ...
    severity: ...
    phase: ...
    span: start..end
symbols:
  ...
hir_summary:
  ...
```

Snapshots summarize stable facts only: diagnostic IDs/severity/phase/span,
symbol rows, and HIR arena counts. They intentionally avoid full JSON dumps.

## Harness Commands

- `nix develop -c just semantic-fixtures`: run fixtures and regenerate
  deterministic snapshots.
- `nix develop -c just semantic-diff`: compare Rust acceptance with the pinned
  PHP reference where available.
- `nix develop -c just verify-phase3`: run the full Phase 3 gate.

Every new diagnostic ID should have at least one fixture or be documented as a
reserved ID in the diagnostic documentation.
